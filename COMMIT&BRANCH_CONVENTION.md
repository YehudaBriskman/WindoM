# Commit Convention

All commits in this repository must follow this format:

---

## Format

```
<type>: <title>

<description>

Generated by [{agent-name}-Agent] [Claude-Bot] of [@Yehuda Briskman]
```

---

## Core Principle (Read This Before Committing)

Commits must be **small, isolated, and logically grouped**.

* Each commit should represent **one clear purpose**
* Do **not mix unrelated changes**
* If you changed multiple things → you need multiple commits
* If you "fixed while adding" → split it

If your commit touches 15 files with different intents — you’re doing it wrong.

---

## Rules

### Title

* Format: `<type>: <title>`
* Length: **5–8 words** (not counting the type prefix)
* Types: `feat`, `fix`, `chore`, `docs`, `refactor`, `style`, `test`
* Use present tense, imperative mood ("add", "fix", "remove")

---

### Description

* **3–8 lines** explaining what changed and why
* Focus on reasoning, not just restating the title
* Each line = full sentence
* No bullet points required

---

### Atomic Commits (Critical Rule)

* One commit = **one responsibility**
* Group only **strongly related changes**
* Never combine:

  * feature + fix
  * refactor + logic change
  * formatting + behavior

Bad:

```
feat: add auth + fix bugs + cleanup code
```

Good:

```
feat: add jwt authentication middleware

fix: resolve token refresh race condition

refactor: simplify auth service structure
```

---

### When You MUST Split Commits

Split if:

* You added a feature and fixed a bug → 2 commits
* You changed logic and formatting → 2 commits
* You touched different domains (auth + UI) → 2 commits
* You’re thinking “eh it’s kinda related” → it’s not

---

### Allowed Grouping (Use Your Brain)

You *can* group changes only if:

* They are part of the same feature flow
* They cannot function independently
* Removing one breaks the whole commit purpose

If you can revert half the commit safely → it should’ve been separate.

---

### Last Line

* Always exactly (with the agent that made the change):

```
Generated by [{agent-name}-Agent] [Claude-Bot] of [@Yehuda Briskman]
```

* Must be separated by a blank line

---

## Example

```
feat: add multi-tab token sync mechanism

Each tab previously refreshed tokens independently causing race conditions.
The first tab rotated the session making others receive unauthorized errors.
This resulted in forced logout across tabs despite valid authentication.
Implemented shared token storage with expiry checks to avoid redundant refreshes.
Tabs now reuse a valid token instead of triggering parallel refresh requests.

Generated by [Frontend-Dev-Agent] [Claude-Bot] of [@Yehuda Briskman]
```

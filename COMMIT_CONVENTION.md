# Commit Convention

All commits in this repository must follow this format:

---

## Format

```
<type>: <title>

<description>

Generated by [Claude-Bot] of [@Yehuda Briskman]
```

---

## Rules

### Title
- Format: `<type>: <title>`
- Length: **5–8 words** (not counting the type prefix)
- Types: `feat`, `fix`, `chore`, `docs`, `refactor`, `style`, `test`
- Use present tense, imperative mood ("add", "fix", "remove" — not "added" or "adds")

### Description
- **3–8 lines** explaining what changed and why
- Focus on the reasoning, not just restating the title
- Each line should be a complete thought
- No bullet points required — plain sentences are fine

### Last Line
- Always the exact string: `Generated by [Claude-Bot] of [@Yehuda Briskman]`
- Must be separated from the description by a blank line

---

## Example

```
fix: prevent logout on multi-tab token refresh race

Each new tab refreshed independently using the same stale cookie.
The first tab rotated the session, causing all others to receive a 401.
This triggered logout across tabs even though auth was still valid.
Added JWT expiry check so tabs reuse a valid stored token without refreshing.
On a failed refresh, the tab now waits briefly and checks storage for a token
stored by another tab before giving up.

Generated by [Claude-Bot] of [@Yehuda Briskman]
```

---

## Git Command Template

```bash
git commit -m "$(cat <<'EOF'
<type>: <5-8 word title>

Line one of description.
Line two of description.
Line three of description.

Generated by [Claude-Bot] of [@Yehuda Briskman]
EOF
)"
```
